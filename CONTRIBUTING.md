## Where can I start?

### Reporting Bugs
- Use a clear and descriptive title for the issue.
- Describe the exact steps which reproduce the problem.
- Explain which behavior you expected to see instead and why.

### Suggesting Enhancements
- Explain why this enhancement would be useful to most users.
- Provide a step-by-step description of the suggested enhancement.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. Follow the project's coding style (enforced by `analysis_options.yaml`).
3. Ensure your code builds without errors (`flutter build windows`).
4. Issue a Pull Request with a clear description of your changes.

## Code of Conduct
Please be respectful to other contributors. We aim to maintain a welcoming and professional environment.
